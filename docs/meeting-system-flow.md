## 消息流向

```
步骤1-4:
  main ──sessions_send──> cppcc-1, cppcc-2, ..., npc-1, npc-2
                          ↓
                        确认回复
                          ↓
  main <──inter_session──  (仅记录，不主动回复)

步骤5-1:
  main ──sessions_send──> cppcc-1, cppcc-2
                          ↓
                        独立分析
                          ↓
  main <──inter_session──  (仅记录)

步骤5-2:
  main ──sessions_send──> cppcc-1, cppcc-2 (含他人意见)
                          ↓
                        立场标记
                          ↓
  main <──inter_session──  (仅记录)

步骤6-1:
  main ──sessions_send──> npc-1, npc-2
                          ↓
                        质询问题
                          ↓
  main <──inter_session──  (仅记录)

步骤6-2:
  main ──sessions_send──> cppcc-1, cppcc-2 (含质询问题)
                          ↓
                        回应质询
                          ↓
  main <──inter_session──  (仅记录)

步骤6-3:
  main ──sessions_send──> npc-1, npc-2
                          ↓
                        投票+理由
                          ↓
  main <──inter_session──  (仅记录)
```

## inter_session 处理规则

```
收到 inter_session 消息时：

1. 识别消息来源
   - provenance.kind === "inter_session"
   - 显示为 [Inter-session message] 前缀

2. 处理动作
   ✅ 记录到会议状态
   ✅ 收集Agent回复
   ❌ 不要主动回复

3. 原因
   - sessions_send 已返回结果
   - inter_session 是系统路由，不需要再响应
   - 避免每条消息触发双重响应
```

## 关键约束

| 约束 | 说明 |
|------|------|
| 严格七步闭环 | 每次会议必须完整执行 |
| 每步用户验收 | 步骤1-4需用户确认后继续 |
| 不跳过步骤 | 禁止从中间步骤开始 |
| inter_session不回复 | 仅记录，不主动响应 |
