# PandaClaw 会议流程系统

> 民主协商决策系统 - 双维度效率控制

---

## 核心设计

### 两个层次

| 层次 | 说明 |
|------|------|
| **大流程** | 文书产生流程（国家文件产生流程） |
| **小流程** | 七步闭环（民主协商决策机制） |

**关系：**
```
大流程（文书产生）
│
├── 步骤1 → 可召开七步闭环会议
├── 步骤2 → 可召开七步闭环会议
├── 步骤3 → 可召开七步闭环会议
└── ...
```

**每个大流程步骤 = 一次七步闭环会议**

---

## 七步闭环（决策机制）

> 详见 AGENTS.md

```
步骤1-4：准备阶段
├── 1. 目标对齐
├── 2. 信息共享
├── 3. 角色分工
└── 4. 协调机制

步骤5：政协协商（两轮）
├── 5-1. 独立输出（同步屏障）
└── 5-2. 意见共享 + 立场

步骤6：人大表决（三轮）
├── 6-1. 质询（强制）
├── 6-2. 政协回应
└── 6-3. 投票 + 理由

步骤7：决策输出
```

## 双维度效率控制

### 维度1：人数（讨论深度）

| 复杂度 | 政协 | 人大 | 说明 |
|--------|------|------|------|
| simple | 2 | 1 | 快速决策 |
| medium | 3 | 2 | 平衡效率 |
| complex | 4 | 3 | 深度审议 |
| enterprise | 5 | 5 | 全会协商 |

### 维度2：验收（严谨度）

| 验收模式 | 步骤1-4处理 | 适用场景 |
|----------|-------------|----------|
| **完整验收** | 每步用户验收 | 关键决策 |
| **关键验收** | 步骤1+3验收 | 常规决策 |
| **跳过验收** | Agent内部确认 | 紧急决策 |

## 大流程定义

### 协商型（白皮书风格）

**大流程：**
```
立项 → 成立起草组 → 调查研究 → 起草初稿 → 征求意见 → 审议表决 → 审定发布
```

**每个步骤可召开七步闭环会议：**
- 立项：是否同意开展？（simple，跳过验收）
- 调查研究：研究方向是否正确？（medium，关键验收）
- 审议表决：是否通过提案？（complex，完整验收）⭐ 关键节点

---

### 战略型（一号文件风格）

**大流程：**
```
中期评估 → 前期研究 → 建议起草 → 征求意见 → 人大审议 → 规划编制 → 最终批准
```

**关键节点（完整验收 + enterprise 人数）：**
- 人大审议
- 最终批准

---

### 决议型（红头文件风格）

**大流程：**
```
立项 → 起草 → 审议 → 签署公布
```

**特点：** 流程短，跳过验收，快速决策

---

### 规划型（规划纲要风格）

**大流程：**
```
中期评估 → 趋势研究 → 建议起草 → 征求意见 → 规划编制 → 审议批准
```

**关键节点：** 审议批准

---

### 立法型（行政法规风格）

**大流程：**
```
立项 → 起草 → 公开征求意见 → 专家论证 → 合规审查 → 审议 → 签署公布 → 施行
```

**特点：** 步骤最多，关键节点完整验收

---

### 纪要型（会议纪要风格）

**大流程：**
```
记录 → 整理 → 确认 → 存档
```

**特点：** 无表决，跳过验收，内部确认

## 核心原则

1. **分形结构** - 大流程每步可召开七步闭环会议
2. **双维度控制** - 人数 + 验收共同决定效率
3. **关键节点** - 高风险、不可逆的步骤需要完整验收
4. **灵活配置** - 根据实际需求组合复杂度和验收模式
