# 会议文件模板指南

> 参照国家官方文件格式，提升会议文档的规范性和正式性

---

## 📕 文件格式类型

### 1. 会议封面

**用途：** 会议文件首页，展示基本信息

**风格：** 参照国务院白皮书封面

**特点：**
- 文件编号：`熊猫协字〔YYYY〕NNN 号`
- 会议议题、类型、复杂程度
- 参会人员列表
- 召开日期、会议状态

**生成命令：**
```bash
node scripts/generate-meeting-document.js meeting-20260327-123456 cover
```

**示例：**
```
┌─────────────────────────────────────────────────────────────────────────┐
│                          民 主 协 商 会 议 文 件                          │
│                              熊 猫 协 字                                  │
│                         熊猫协字〔2026〕001 号                            │
├─────────────────────────────────────────────────────────────────────────┤
│   会议议题：开发 AI 智能客服系统                                          │
│   会议类型：项目规划会议                                                 │
│   复杂程度：复杂（4 政协 +3 人大）                                        │
│   召开日期：2026/03/27                                                  │
│   会议状态：已完成                                                      │
├─────────────────────────────────────────────────────────────────────────┤
│   政协委员：cppcc-1、cppcc-2、cppcc-3、cppcc-4                           │
│   人大代表：npc-1、npc-2、npc-3                                         │
└─────────────────────────────────────────────────────────────────────────┘
```

---

### 2. 完整报告（白皮书风格）

**用途：** 正式会议记录，存档备案

**风格：** 参照国务院白皮书

**特点：**
- 目录结构清晰
- 章节编号规范（一、二、三... (一)(二)(三)... 1.2.3...）
- 完整的协商过程记录
- 表决结果和决议
- 附件清单

**生成命令：**
```bash
node scripts/generate-meeting-document.js meeting-20260327-123456 full
```

**目录结构：**
```
一、会议概况
    （一）会议议题
    （二）会议背景
    （三）参会人员
    （四）身份配置

二、协商过程
    （一）目标对齐（步骤 1）
    （二）信息共享（步骤 2）
    （三）角色分工（步骤 3）
    （四）协调机制（步骤 4）
    （五）政协协商（步骤 5）
        1. 独立输出（5-1）
        2. 意见共享（5-2）
    （六）人大表决（步骤 6）
        1. 质询环节（6-1）
        2. 政协回应（6-2）
        3. 投票表决（6-3）

三、会议决议
    （一）表决结果
    （二）最终决策
    （三）后续工作安排

四、附件
    （一）政协委员意见书
    （二）人大代表质询记录
    （三）投票记录明细
```

### 3. 红头文件

**用途：** 正式决议发布，下发执行

**风格：** 参照中共中央红头文件

**特点：**
- 红头边框（ASCII 艺术）
- 文件编号
- 正式决议文本
- 主题词、抄送单位
- 印发日期

```bash
node scripts/generate-meeting-document.js meeting-20260327-123456 redheader
```

### 4. 会议纪要

**用途：** 快速浏览，会议摘要

**风格：** 简洁明了

**特点：**
- 基本信息
- 协商意见摘要
- 质询问题摘要
- 表决结果
- 决议结论

**生成命令：**
```bash
node scripts/generate-meeting-document.js meeting-20260327-123456 minutes
```

**示例：**
```
【会议纪要】
```

## 🚀 使用方法

### 方式 1：命令行生成

```bash
# 查看所有格式
node scripts/generate-meeting-document.js

# 生成指定会议的报告
node scripts/generate-meeting-document.js meeting-20260327-123456 full

# 生成红头文件
node scripts/generate-meeting-document.js meeting-20260327-123456 redheader

# 生成会议纪要
node scripts/generate-meeting-document.js meeting-20260327-123456 minutes
```

### 方式 2：代码调用

```javascript
const { generateDocument, saveDocument } = require('./scripts/meeting-state-store');

// 生成文档内容
const doc = generateDocument('meeting-20260327-123456', 'full');

// 保存到文件
const filepath = saveDocument('meeting-20260327-123456', 'full');

// 保存到自定义目录
const customPath = saveDocument('meeting-20260327-123456', 'redheader', './output');
```

## 📊 文件编号规则

**格式：** `熊猫协字〔YYYY〕NNN 号`

- `熊猫协字`：PandaClaw 协商文件
- `〔YYYY〕`：年份
- `NNN`：序号（001-999）

**示例：**
- `熊猫协字〔2026〕001 号`
- `熊猫协字〔2026〕123 号`

```
│   ├── meeting-xxxx.json

│       ├── meeting-xxxx-full.md
│       ├── meeting-xxxx-redheader.md
│       └── meeting-xxxx-minutes.md
```

## 🎯 最佳实践

### 1. 会议结束后立即生成

```javascript
// 会议完成后自动生成文档
meeting.completeMeeting(decision);
saveDocument(meeting.id, 'full');
saveDocument(meeting.id, 'redheader');
```

### 2. 归档重要会议

```bash
# 将红头文件归档到项目文档
cp meetings/documents/meeting-xxx-redheader.md ../project-docs/decisions/
```

### 3. 定期备份

```bash
# 备份所有会议文档
tar -czf meetings-backup-$(date +%Y%m%d).tar.gz meetings/documents/
```

## 📝 自定义模板

如需自定义模板格式，修改：

```
scripts/meeting-document-template.js
```

可自定义：
- 封面样式
- 章节标题
- 边框风格
- 字体（导出为 PDF 时）
